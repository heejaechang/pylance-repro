# Issue #7412 - Find All References doesn't ignore `__editable__` directories

**Issue**: https://github.com/microsoft/pylance-release/issues/7412

## Reproduction

This workspace mimics the scenario when `pip install -e . --config-settings editable_mode=strict` creates `__editable__.modulename.xxx` directories in site-packages.

### Scenario

The user has a project (`pegen`) installed in strict editable mode. Pip creates auto-generated finder files in `site-packages/__editable__.pegen.0.3.1/` that re-export symbols from the source.

### Repro Steps (from issue)

1. Open this workspace in VS Code with Pylance
2. Open `src/pegen/grammar.py`
3. Right-click on `GrammarVisitor` class name
4. Select "Find All References"

### Expected (from issue reporter)

> References should NOT include files from `__editable__.pegen.0.3.1` directories since they are auto-generated by pip.

### Actual (from issue reporter)

> "Find All References" includes results from `__editable__.modulename.xxx` directories.

References include `site-packages/__editable__.pegen.0.3.1/pegen/__init__.py` which is an auto-generated file.

## Verification Checklist

- [ ] "Find All References" on `GrammarVisitor` in `src/pegen/grammar.py` does NOT include results from `__editable__` directories
- [ ] References from normal user code (`src/pegen/parser.py`) still appear correctly

## Workspace Structure

- `src/pegen/grammar.py` - User source with `GrammarVisitor` class (the target symbol)
- `src/pegen/parser.py` - User source that references `GrammarVisitor`
- `site-packages/__editable__.pegen.0.3.1/pegen/__init__.py` - Auto-generated editable install file (should be excluded from references)
- `.vscode/settings.json` - Adds both `src` and `site-packages` to extraPaths so Pylance sees all files

## Notes

- The `.vscode/settings.json` adds `site-packages` to `extraPaths` to simulate the scenario where Pylance tracks files in the editable install directory.
- In a real scenario, these paths would be discovered through the Python environment's site-packages.
